## 1. 项目概述

### 1.1 项目愿景
面向大一软件工程学生设计的智能知识管理和复习工具，通过AI辅助帮助学生高效管理教材、整理知识结构、管理作业任务，并基于遗忘曲线进行智能复习提醒。

### 1.2 核心价值主张
- **结构化知识管理**: 树状结构 + 标签补充，让知识脉络清晰可见
- **智能复习**: 基于掌握程度的学习模式，精准推送复习内容
- **作业追踪**: 全流程作业管理，从录入到掌握程度追踪
- **AI助手**: DeepSeek API 驱动的答疑和答案生成

### 1.3 目标用户画像
| 维度 | 描述 |
|------|------|
| 年级/专业 | 大一软件工程学生 |
| 核心需求 | 课程知识整理、作业管理、考试复习 |
| 技术能力 | 具备基础编程能力，能使用命令行 |
| 使用场景 | 晚间复习、周末整理、考试周冲刺 |
| 痛点 | 知识碎片化、复习无重点、作业遗忘 |

**数据模型**:
```python
KnowledgeNode {
    id: int (PK)
    book_id: int (FK)
    parent_id: int (FK, nullable)  # 自关联实现树状

```

#### 2.2.4 学习模式

| 功能 | 描述 | 优先级 |
|------|------|--------|

    id: int (PK)
    node_id: int (FK)

    review_count: int

#### 2.2.5 复习模式

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 考试范围设置 | 指定复习的教材章节 | P0 |
| 复习清单生成 | 基于遗忘曲线排序 | P1 |
| 智能推送 | 优先推送掌握度低的知识点 | P1 |
| 复习统计 | 今日/本周复习数据 | P1 |
| 掌握度报表 | 各知识点掌握度可视化 | P2 |

**复习优先级算法**:
```
优先级 = 基础分 × 遗忘因子 × 重要因子

遗忘因子 = e^(-(当前时间-最后复习时间)/半衰期)
基础分 = 1 - (正确次数/总尝试次数)
重要因子 = 1 + (标签中"重要"数量 × 0.2)
```

### 3.3 数据流设计

#### 3.3.1 PDF教材录入流程
```
用户上传PDF
    ↓
后端接收文件 → 保存至 /data/uploads/
    ↓
启动PDF解析任务 (异步，避免阻塞)
    ↓
PyMuPDF 逐页读取 (流式，限制内存)
    ↓
提取目录结构 → 构建知识节点树
    ↓
存储解析结果 → 更新Book.extracted_text
    ↓
返回解析结果给前端
```

#### 3.3.2 AI问答流程
```
用户提问
    ↓
检查本地缓存 (基于node_id + 问题hash)
    ↓
缓存命中 → 直接返回
    ↓
缓存未命中 → 调用DeepSeek API
    ↓
响应存入缓存 → 返回给前端
    ↓
记录Conversation
```

### 3.4 1核1G服务器优化策略

#### 3.4.1 内存优化
| 策略 | 实现方式 | 预期效果 |
|------|----------|----------|
| PDF流式处理 | 使用`fitz.open(stream=...)`逐页读取 | 内存峰值<100MB |
| pandas按chunk | `pandas.read_csv(chunksize=100)` | 内存峰值<50MB |
| 懒加载 | 前端分页加载，后端limit/offset | 减少单次内存占用 |
| 缓存限制 | AI缓存文件大小限制(1MB/条) | 缓存总大小<200MB |
| 数据库连接池 | 单连接复用 | 减少连接开销 |

#### 3.4.2 CPU优化
| 策略 | 实现方式 |
|------|----------|
| PDF解析后台化 | 使用Flask异步任务或线程池 |
| AI请求限流 | 限制并发请求数≤3 |
| 缓存预热 | 只缓存高频访问内容 |

#### 3.4.3 存储优化
| 策略 | 实现方式 |
|------|----------|
| PDF不重复存储 | 同一文件hash去重 |
| 缓存定期清理 | 超过7天的AI缓存自动删除 |
| 日志轮转 | 使用rotating log handler |

**PDF解析伪代码**:
```python
def parse_pdf(pdf_path, book_id):
    doc = fitz.open(pdf_path)
    chapters = extract_toc(doc)  # 提取目录
    
    for chapter in chapters:
        node = KnowledgeNode(
            book_id=book_id,
            title=chapter.title,
            level=chapter.level,
            page_start=chapter.start_page,
            page_end=chapter.end_page
        )
        db.session.add(node)
        
        # 每处理100页提交一次，避免内存累积
        if page_count % 100 == 0:
            db.session.commit()
    
    db.session.commit()
    doc.close()
```

## 8. 风险管理

### 8.1 风险识别

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| PDF解析内存溢出 | 高 | 高 | 流式处理 + 分批提交 |
| DeepSeek API延迟高 | 中 | 中 | 本地缓存 + 降级策略 |
| 1核服务器性能不足 | 中 | 高 | 优化算法 + 限流 |
| CSV导入大数据量崩溃 | 中 | 中 | 分批处理 + 进度反馈 |
| 数据丢失 | 低 | 极高 | 自动备份 + JSON导出 |

### 8.2 应急响应

| 场景 | 响应措施 |
|------|----------|
| 服务宕机 | 自动重启脚本 + 监控告警 |
| 内存不足 | OOM自动重启 + 日志分析 |
| 数据库损坏 | JSON备份恢复 |
| API超时 | 重试机制 + 降级提示 |

| NOT_FOUND | 资源不存在 |
| DATABASE_ERROR | 数据库操作失败 |
| PDF_PARSE_ERROR | PDF解析失败 |
| AI_SERVICE_ERROR | AI服务调用失败 |
| SYNC_ERROR | 同步操作失败 |
| FILE_TOO_LARGE | 文件超过限制 |
